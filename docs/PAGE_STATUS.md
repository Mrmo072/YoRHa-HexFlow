# 页面实现状态

本文件由 `frontend/src/config/pageStatus.json` 生成，用于作为页面导航、占位说明和文档状态的统一参考。

## 已落地主链路

- `协议定义` (/protocol)
- `指令管理` (/instruction)
- `指令加工` (/processing)
- `编排绑定` (/orchestration)

## 占位或待实现页面

- `通讯调试` (/terminal)
- `数据中心` (/datahub)

---

## 协议定义 / Protocol Definition

- 路径: `/protocol`
- 快捷键: `A`
- 当前状态: 已接入 SQLite / FastAPI 主链路
- 摘要: 协议定义页用于维护协议外壳、容器结构、固定块、长度块、校验块和插槽块，当前已经接入仓库内 SQLite 与 FastAPI 协议接口。

### 已具备
- 协议列表已通过 SQLite 持久化。
- 支持协议新增、删除、块编辑与嵌套容器导航。
- 协议修改会通过防抖保存同步到后端。

### 后续建议
- 继续补协议块拖拽、块属性编辑和异常保存链路的交互测试。
- 逐步收敛协议块类型和属性定义，减少页面内硬编码。

---

## 指令管理 / Instruction Management

- 路径: `/instruction`
- 快捷键: `B`
- 当前状态: 已接入 SQLite / FastAPI 主链路
- 摘要: 指令管理页用于维护指令元信息、模块模板编排和积木属性，当前已经接入 SQLite 指令与算子模板接口。

### 已具备
- 支持指令新增、删除、保存、重置和本地未保存状态提示。
- 模块模板支持加载态、错误态和重试。
- 指令字段已通过统一 payload 规范化后再写回后端。

### 后续建议
- 继续减少页面与 hook 之间的隐式约定。
- 补更细的块移动、字段引用和保存失败恢复测试。

---

## 指令加工 / Instruction Processing

- 路径: `/processing`
- 快捷键: `R`
- 当前状态: 已接入共享指令数据链路
- 摘要: 指令加工页用于基于现有指令定义填写参数并预组装 payload，当前已经复用共享指令数据状态。

### 已具备
- 与指令管理页共享同一份指令列表状态。
- 支持按当前指令结构渲染参数填写表单。
- 支持打开日期选择器并预览发送 payload。

### 后续建议
- 补发包结果展示与失败态。
- 将参数渲染规则从页面组件继续下沉为可测试配置。

---

## 编排绑定 / Orchestration Binding

- 路径: `/orchestration`
- 快捷键: `C`
- 当前状态: 已接入数据库协议与指令预览
- 摘要: 编排绑定页用于将协议壳与指令内核组合成完整报文结构，当前已经读取数据库中的协议与指令并支持插槽注入预览。

### 已具备
- 支持选择协议壳和指令内核进行合成预览。
- 支持按插槽规则将指令结构注入协议壳。
- 提供结构预览和基础长度统计。

### 后续建议
- 考虑将绑定关系持久化而不是只保留页面内临时状态。
- 补总长度、插槽缺失和边界结构的回归测试。

---

## 通讯调试 / Communication Terminal

- 路径: `/terminal`
- 快捷键: `D`
- 当前状态: 占位页 / 未接入串口或 TCP 调试链路
- 摘要: 当前项目的稳定主链路集中在协议定义、指令管理、指令加工和编排绑定。通讯调试页还没有接入真实会话、发送记录、串口枚举或报文往返结果，所以这里明确展示实现边界。

### 已具备
- 可以在“指令加工”页完成参数填写与 payload 预组装。
- 可以在“编排绑定”页预览协议壳与指令内核合成后的结构。
- 后端当前没有提供串口、TCP、WebSocket 调试接口。

### 后续建议
- 补通讯配置模型：串口参数、目标地址、发送模式。
- 增加发送历史、原始报文、响应报文与错误日志面板。
- 根据后端能力决定接串口库或 TCP 调试 API。

---

## 数据中心 / Data Hub

- 路径: `/datahub`
- 快捷键: `E`
- 当前状态: 占位页 / 未接入导入导出与配置治理
- 摘要: 数据中心页目前还没有接上真正的导入、导出、备份、环境切换和批量配置能力。现阶段协议、指令和算子模板都已经落在 SQLite 与 API 上，但围绕这些数据的工程化操作台还没有独立实现。

### 已具备
- 协议、指令和算子模板已经由 SQLite 持久化。
- 项目支持通过后端 API 读取与修改协议、指令数据。
- 当前仍缺少面向业务用户的可视化导入导出入口。

### 后续建议
- 补 JSON 导入导出与示例包下载。
- 增加数据库备份、恢复和环境状态展示。
- 将 README 中的能力描述与页面实现状态持续保持一致。
